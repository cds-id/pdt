Connection Manager Design

Lifecycle

On server start → load all wa_numbers with status=connected → reconnect each. On shutdown → graceful disconnect all. On crash → auto-reconnect with backoff.

Multi-Number

Each number gets its own whatsmeow Client instance running in a goroutine. Connection Manager holds a map[uint]Client keyed by wa_number ID.

Recovery

If a connection drops, exponential backoff retry (5s → 10s → 30s → 60s max). After 5 failures, mark status=disconnected and notify user via WebSocket.

Sender Worker

Background goroutine polls wa_outbox for approved messages every 5s. Sends via the appropriate whatsmeow client. Updates status to sent/failed.
