Section 5: Weaviate Integration & Embedding Pipeline

Vector search architecture for context-aware chat analytics

Embedding Pipeline Flow
New message saved to wa_messages
        │
        ▼
Push to embedding queue (Go channel, buffered)
        │
        ▼
Embedding Worker (goroutine)
        │
        ├─ Batch messages (up to 50 or 5s timeout)
        │
        ▼
Call Weaviate's vectorizer module
(text2vec-transformers or text2vec-openai)
        │
        ▼
Weaviate stores vector + metadata:
  ┌────────────────────────────────────┐
  │  WaMessageEmbedding                │
  │                                    │
  │  message_id  → FK to wa_messages   │
  │  listener_id → for filtering       │
  │  user_id     → for isolation       │
  │  content     → original text       │
  │  sender_name → who sent it         │
  │  timestamp   → when sent           │
  │                                    │
  │  vector     → auto-generated       │
  └────────────────────────────────────┘
        │
    On failure → retry queue (3 attempts)
    After 3 fails → log, skip (MySQL is source of truth)

Search & Query Patterns

Semantic Search

Agent's semantic_search tool sends natural language query to Weaviate nearText. Filters by user_id + optional listener_id + date range. Returns top-K messages ranked by similarity.

Summarization Context

For summarize_chat, agent first fetches messages from MySQL (time range), then optionally uses Weaviate to find related messages outside the range for context continuity.

Hybrid Search

Weaviate supports hybrid (keyword + vector) out of the box. Agent can use search_messages (MySQL keyword) for exact matches and semantic_search (Weaviate) for meaning-based — or combine results.

Backfill Strategy

On first setup or if Weaviate is rebuilt: background job reads all wa_messages from MySQL and re-embeds them in batches. Progress tracked in logs.

Resilience Design

  • MySQL is source of truth — Weaviate is a search index, not primary storage. If Weaviate goes down, messages keep flowing into MySQL. Vector search degrades gracefully (agent falls back to keyword search).
  • Async embedding — message ingestion is never blocked by Weaviate latency. Buffered Go channel decouples the two.
  • Idempotent upserts — embedding worker uses message_id as deterministic UUID in Weaviate, safe to retry.
  • Health check — Weaviate client pings on startup. If unavailable, logs warning, disables vector features, retries periodically.
